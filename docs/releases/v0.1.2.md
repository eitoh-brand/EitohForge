# EitohForge v0.1.2 Release Notes

This release focuses on **PyPI/readme quality** and **release pipeline reliability**. No runtime/SDK breaking behavior is intended.

## Highlights

- Expanded `README.md` into a **PyPI-grade, long-form** guide with:
  - a clear gist/overview for first-time readers
  - CLI usage reference and examples
  - module-by-module SDK usage examples
  - a third-party support matrix
  - a multi-page docs map into `docs/guides/`
- Improved CLI help output:
  - `eitohforge create project|crud` now shows better descriptions for required arguments
  - `eitohforge version` is now sourced from installed package metadata (with a safe dev fallback)
- CI/publishing reliability improvements so “publish” flows succeed consistently:
  - security-compliance audit runs against exported locked requirements (excluding editable/local package issues)
  - license policy parsing fixes so LGPL libraries are handled correctly
  - SBOM/license tooling steps execute via `uv run`
  - build provenance attestations are skipped for user-owned **private** repositories (where GitHub cannot persist attestations)

## Included Scope

- Documentation & packaging polish (PyPI long description)
- CLI UX/ergonomics for `--help` and `version`
- Release pipeline gates (CI security + packaging/provenance steps)
