# EitohForge v0.1.5 Release Notes

CLI expansion, operational helpers, README narrative cleanup, and OpenAPI/Swagger documentation. No intentional breaking changes for existing SDK consumers.

## Highlights

- **CLI**: New top-level groups `config`, `docs`, `ops`, `feature-flags`, `doctor`; `config feature-list` / `feature-set` for common `EITOHFORGE_*` flags; `create pseudocode` for phased implementation notes.
- **`eitohforge ops`**: `check` and `smoke` probe `/health`, `/ready`, `/status`, `/sdk/capabilities`, and `GET /openapi.json` (compact summary on check).
- **README**: Progressive reading order (REST first → toggles with wire tables in one section → data/env → scale → CLI → SDK); Gist and Quick Start cover **FastAPI OpenAPI/Swagger** (`/docs`, `/redoc`, `/openapi.json`) and production hardening notes.
- **Docs**: `docs/guides/usage-complete.md` mentions interactive API docs.

## Validation

- `uv run pytest --no-cov tests/unit/test_cli.py`
- `python -m build` + `twine check dist/*`

## Upgrade notes

- No code migration required for generated apps; reinstall the CLI to pick up new commands.
