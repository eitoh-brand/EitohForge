# EitohForge v0.1.6 Release Notes

Platform primitives for repository contracts, feature flag persistence, inbound webhooks, cloud secrets, and provider scaffolding.

## Highlights

- **Repositories & DTOs**: export `BaseRepository` in the domain layer; add `ok` / `paginated` / `err` helpers around `ApiResponse` / `PaginatedApiResponse` / `ApiErrorResponse` (optional sugar, envelopes are unchanged).
- **Feature flags**: JSON mappers + `FeatureFlagService.reload` and `load_definitions_from_redis_json` for shared Redis-backed flag definitions.
- **Secrets**: `build_secret_provider` now supports `EITOHFORGE_SECRET_PROVIDER=aws|azure` (AWS Secrets Manager via optional `boto3` extra; Azure Key Vault via REST + bearer token) in addition to existing env/Vault.
- **Webhooks & notifications**: inbound HMAC-verifying webhook router (`register_inbound_webhook_router`) and a SendGrid email sender (`build_sendgrid_email_sender`) wired through the notification gateway.
- **Registries & policies**: `PolicyRegistry` for named ABAC policies and `EngineRegistry` for named SQLAlchemy engines (complements the existing database role registry).
- **CLI**: `eitohforge create provider <name>` scaffolds an integration provider stub under `app/providers/`.

## Validation

- `uv run pytest -q`
- `uv run ruff check src tests`
- `python -m build` + `twine check dist/*`

## Upgrade notes

- No breaking API changes are intended; these are additive primitives. Existing projects can ignore the new helpers until needed.
